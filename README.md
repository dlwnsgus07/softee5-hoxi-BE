## Branch Strategy

간략화된 Git-Flow 전략을 사용합니다. (참고: https://techblog.woowahan.com/2553/)

- `main` : 제품으로 출시된 브랜치
    - `dev`: 개발용 브랜치
        - `feature/branch-name` : 각 기능별 브랜치
- `fix`: 출시 버전에서 발생한 버그수정 브랜치

## Commit Message Convention

```
type : commit message
```

모든 커밋 메세지는 명령문 (현재시제) 를 사용하여 작성합니다. 모든 커밋 메세지 앞에는 다음의 `type` 을 작성합니다.

- `feat`: 새로운 기능 추가
- `fix`: 버그수정
- `refactor`: 코드 리팩토링
- `chore`: 기타 작업
